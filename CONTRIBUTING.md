# Contributing

Thank you for your interest in VaultSecure. This repository is primarily maintained as an academic and portfolio project, but contributions and suggestions are welcome.

## Contribution Guidelines

Before submitting a change:

1. Fork the repository.
2. Create a feature branch.
3. Keep changes focused and clearly documented.
4. Run the test suite with `pytest`.
5. Avoid committing secrets, credentials, local databases, or environment-specific files.

## Security-Focused Contributions

Security-related contributions are especially welcome, including improvements to:

- Key management
- Encryption design
- Password hashing and verification
- MFA workflows
- Test coverage
- Dependency management
- Secure coding practices

For sensitive security findings, follow the process in [SECURITY.md](SECURITY.md) instead of opening a public issue.
